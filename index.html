1 AI xx 系统主要是做什么的？

这是一个面向业务场景的 AI 能力系统，核心是把用户问题、业务数据、知识库和工具调用串起来，提供问答、检索、任务执行、内容生成或流程辅助能力。架构上通常包含入口层、意图识别层、RAG/Agent 编排层、工具层、模型层和监控评估层。

2 LangChain / LangGraph 主要用了什么 AI 技术？

LangChain 主要用于 LLM 调用封装、Prompt 模板、工具调用、RAG 链路和记忆管理；LangGraph 更偏状态机/图编排，适合多步骤 Agent、条件分支、循环、人工介入和可恢复执行。底层涉及 Function Calling、ReAct、RAG、Embedding、向量检索、状态管理等技术。

3 有没有了解过 Manus 这种通用 Agent？

了解。通用 Agent 的目标是把规划、执行、工具调用、浏览器/文件/代码操作等能力统一起来，让模型可以完成复杂任务。它的关键不只是模型能力，还包括任务拆解、环境交互、执行反馈、错误恢复和上下文治理。

4 这些通用 Agent 一般分几层功能？

一般可以分为：用户交互层、任务规划层、记忆/上下文层、工具执行层、反馈评估层、权限与安全层、运行时/沙箱层。更工程化的系统还会有日志、成本、队列、监控和审计。

5 有没有构建过多智能体架构？

可以回答：做过或参与过类似设计。通常会把复杂任务拆成规划 Agent、检索 Agent、执行 Agent、校验 Agent、总结 Agent 等角色，通过路由器或状态图协调，避免一个 Agent 同时承担太多职责。

6 什么是多 Agent？

多 Agent 是把一个复杂 AI 任务拆给多个具备不同职责的智能体协同完成。每个 Agent 可以有自己的 Prompt、工具权限、上下文和输出规范。

7 为什么需要多 Agent？

因为复杂任务往往涉及规划、检索、计算、工具调用、校验等多个环节。多 Agent 可以降低单个 Prompt 的复杂度，让职责更清晰，也更方便调试、评估和扩展。

8 什么场景下需要多 Agent 架构？

适合长链路、多工具、多角色、多阶段审核的任务，比如报告生成、代码修复、数据分析、企业知识问答加流程办理、售前方案生成等。如果只是简单问答，不一定需要多 Agent。

9 单个 Agent 也可以分配多个职能，为什么还要多 Agent？

单 Agent 可以做，但随着任务复杂度上升，Prompt 会变长、职责会混乱、错误定位困难。多 Agent 的价值在于解耦职责、控制工具权限、隔离上下文，并提升可观测性。

10 多 Agent 主要是为了解决什么问题？

主要解决复杂任务拆解、职责隔离、长链路稳定性、工具权限控制、结果校验和可维护性问题。

12 Agent 使用量是多少？

按真实数据回答。可以说：日均会话 X，Agent 任务 X，工具调用 X，主要使用场景是 知识问答/文档生成/业务办理。

13 Token 消耗量大概多少？

按真实数据回答。建议分输入、输出、Embedding、重排或长上下文任务说明，例如：日均约 X tokens，高峰约 X，其中 RAG 检索上下文占比较高。

14 有没有考虑怎么节约 AI 成本？

有。成本优化主要从模型分层、缓存、Prompt 压缩、RAG 召回控制、上下文裁剪、批处理、失败重试控制和重复任务去重几个方向做。

15 做了哪些成本优化方案？

常见方案包括：简单问题走小模型，复杂问题走大模型；Embedding 和问答结果缓存；减少 TopK 和无效上下文；对历史会话摘要压缩；工具调用结果复用；任务幂等和去重；监控单次会话 Token 上限。

16 最终降了多少成本，收益多少？

按真实数据回答。建议表达为：通过模型分层和缓存后，单次任务平均 Token 降低 X%，模型调用成本降低 X%；收益可从人工处理时长下降、响应速度提升、用户自助解决率提升来说明。

1 Claude Code 的多层架构怎么做？

可以按“交互层、任务规划层、上下文管理层、工具执行层、沙箱/权限层、模型调用层、日志监控层”来讲。核心是模型循环读取上下文，决定下一步工具调用，执行后把结果回灌，再继续规划。

2 PS 网页版实现方案？

前端可以用 Canvas/WebGL 渲染画布，图层系统管理图片、文字、形状和滤镜；复杂图像处理可用 WebAssembly 或 WebGL Shader；状态管理维护历史栈、选区、缩放、拖拽和导出。

3 Canvas 无限画布怎么实现？

本质是维护一个世界坐标系，通过相机偏移和缩放把世界坐标映射到屏幕坐标。只渲染视口内元素，配合空间索引、分层缓存、离屏 Canvas 和事件坐标反算优化性能。

4 WebAssembly 了解吗？

WebAssembly 是浏览器里的高性能二进制执行格式，适合把 C/C++/Rust 的图像处理、音视频、压缩、加密等计算密集逻辑迁到前端运行。

5 Docker 和 K8S 的关系？

Docker 负责镜像构建和容器运行，K8S 负责容器编排、调度、服务发现、扩缩容、滚动发布和故障恢复。简单说 Docker 是容器能力，K8S 是大规模管理容器的平台。

6 容器和镜像关系，怎么挂载？

镜像是只读模板，容器是镜像运行后的实例。挂载可以用 bind mount、volume、ConfigMap、Secret 等方式，把宿主机目录或外部存储映射到容器路径。

7 K8S 多个服务怎么通讯？

同集群内通常通过 Service 名称做 DNS 访问，比如 http://user-service.namespace.svc.cluster.local。服务间可以通过 ClusterIP、Headless Service、Ingress、Service Mesh 等方式通信。

8 怎么进行团队管理？

先拆清目标、边界和优先级，再把任务拆成里程碑和负责人。过程里关注风险、依赖、代码质量和沟通节奏，用周会/看板/评审保证信息透明。

9 怎么推动项目进度？

我会先明确交付标准和关键路径，然后拆任务、定排期、识别阻塞点。推进中每天跟进风险，对跨团队依赖提前沟通，必要时做范围裁剪保证核心目标上线。

10 Express 底层原理？如果设计怎么做？

Express 核心是基于 Node HTTP 模块封装请求响应对象，再通过中间件栈按顺序匹配和执行。自己实现时会设计路由表、中间件队列、next() 调度、错误中间件和请求上下文。

11 入职后如何快速上手业务需求？

先读业务文档和核心流程，再跑通本地环境和主链路；接着看数据库表、接口、日志和线上监控。前几天优先修小需求或 Bug 建立手感，再逐步承担核心模块。

12 职业中比较有挑战性的问题？

可以选一个真实项目讲：比如 AI 系统从 0 到 1 落地、RAG 召回准确率提升、长任务稳定性、性能优化或跨团队推进。回答重点放在问题背景、你的判断、具体动作和结果。

1 AI 项目架构情况？

可以按入口层、业务编排层、AI 能力层、知识库层、工具服务层、数据存储层和监控评估层来讲。请求进入后先做鉴权和意图识别，再选择 RAG、Agent 或普通模型问答链路。

2 为什么选择 LangChain、LangGraph？

LangChain 生态成熟，适合快速接入模型、Prompt、工具和 RAG；LangGraph 适合有状态、多分支、可循环的 Agent 工作流。选型依据是开发效率、可扩展性、社区成熟度和复杂任务编排能力。

3 抽象通用 AI 能力模块做了什么？

一般会抽象模型调用、Prompt 模板、工具注册、RAG 检索、会话上下文、流式输出、错误重试、日志埋点、成本统计和权限控制，业务只需要组合这些能力。

4 AI 调用工具失败怎么解决？

先区分是参数错误、权限错误、工具超时、服务异常还是模型选择错误工具。工程上会做参数 schema 校验、重试、降级、错误反馈给模型重新规划、人工兜底和详细日志。

5 RAG 流程？

文档解析后切分成 chunk，做清洗、元数据标注和 Embedding 入库；用户提问时做查询改写/向量化，召回相关片段，必要时重排，然后把上下文拼进 Prompt 生成答案并附来源。

6 怎么加速 RAG 检索？

可以用向量索引、TopK 控制、元数据过滤、分库分集合、缓存热门 Query、异步预处理、混合检索和重排分层。大规模时还要优化 chunk 粒度和索引参数。

7 向量库数据量？

按真实数据回答。建议说清楚文档数量、chunk 数量、向量维度和增量更新频率。

8 向量库用什么，选型依据？

常见选择有 Milvus、pgvector、Elasticsearch/OpenSearch、Chroma、Qdrant。依据包括数据规模、过滤能力、部署成本、混合检索能力、团队运维经验和与现有数据库的集成。

9 大模型选型逻辑？

从效果、成本、延迟、上下文长度、工具调用能力、结构化输出、数据合规和稳定性评估。通常采用模型分层：简单任务小模型，复杂推理或生成任务大模型。

10 政策文件场景怎么确认召回准确性？

需要保留来源、文件名、条款编号、发布时间等元数据；召回后做重排和引用校验。评估上可以建标准问答集，统计召回率、命中率、答案引用准确率，并加入人工抽检。

11 上下文压缩策略？

常用策略是历史会话摘要、按轮次保留最近上下文、按重要性提取关键信息、工具结果结构化缓存、RAG 片段去重和长度预算控制。

12 压缩质量有问题怎么解决？

可以加结构化摘要模板、关键信息字段校验、保留原文索引、重要内容不压缩、用更强模型做压缩，或在回答前做压缩结果校验。关键业务场景要允许回查原始上下文。

13 Claude Code 泄露事件 / 源码研究？

可以谨慎回答：关注过相关讨论，但不会依赖非官方泄露内容。更关注公开可验证的架构思想，比如上下文治理、工具权限、执行循环和沙箱安全。

14 Claude Code 上下文治理？

核心是把用户指令、项目文件、工具结果、历史摘要和系统约束分层管理，并控制进入模型的上下文预算。它会按相关性选择文件和历史，不是简单把全部内容塞进去。

15 Claude Code 核心循环流程？

用户提出目标后，模型读取上下文并规划下一步；选择工具读取文件、搜索、编辑或执行命令；工具结果返回后模型继续判断，直到完成目标或遇到阻塞。

16 多租户 K8S 怎么部署，一个租户一个 Pod 吗？

不一定。小规模可以共享服务，通过租户 ID 做数据隔离；高隔离要求可以按租户 namespace、独立数据库、独立实例或独立 Pod。选择取决于安全、成本、资源隔离和运维复杂度。

1 企微 H5 登录流程？

前端跳转企业微信 OAuth 授权页，带 appid、redirect_uri、scope、state；回调拿到 code 后给后端，后端用 code 换取用户身份，再生成业务 token 返回前端。

2 企微只能绑定两个域名，多个业务域名怎么处理？

可以统一用一个授权中转域名做 OAuth 回调，再根据 state 或业务参数跳回实际业务域名。也可以通过网关/反向代理统一入口。

3 没有设计稿怎么推进？

先确认用户目标、核心流程和字段，再快速画低保真原型或可交互 Demo 给业务确认。实现时优先复用现有组件库，保证交互一致，避免陷入视觉细节。

4 做过 H5 用户打标签吗？

可以通过用户行为、来源渠道、企微身份、表单选择、浏览路径等维度打标签。标签一般写入用户画像表或埋点系统，用于运营分群和后续触达。

5 用户进入页面后怎么获取唯一 ID？

登录态下使用业务用户 ID 或企微 userId；未登录时可以用设备指纹、匿名 ID、cookie/localStorage 生成临时 ID，登录后再做匿名 ID 和用户 ID 绑定。

6 企微回调里的 ID 同步怎么做？

后端拿到企微 userId/open_userid 后，与业务用户表建立映射关系。如果用户未注册，则创建或进入绑定流程；已存在则更新企微信息和最后登录时间。

7 埋点怎么做，获取哪些数据？

埋点分曝光、点击、提交、停留、错误和性能。字段包括用户 ID、页面、事件名、时间、渠道、设备、业务参数、会话 ID 等，敏感信息要脱敏。

8 RAG 检索怎么做？

文档切分入库后，问题经过改写和向量化，先做向量召回或混合检索，再按相似度、业务标签、时间、权限过滤，最后重排并交给模型生成答案。

9 海豚同步 MySQL IO 爆红怎么排查？

先看是源库读 IO 还是目标库写 IO，高峰是否和调度时间重合；检查三表 Join 是否走索引、是否全表扫描、批量大小、事务大小、目标表索引过多、磁盘和 binlog 压力。优化可做增量同步、分页批量、加索引、拆任务、降低并发、错峰执行、先落临时表再合并。

10 MySQL 大表慢插入怎么优化？

检查索引数量、唯一约束、事务大小、磁盘 IO、锁等待和自增主键热点。方案包括批量插入、减少不必要索引、分区/分表、异步队列削峰、冷热数据拆分、调整参数和归档历史数据。

1 数据库和缓存一致性？

常用策略是先更新数据库，再删除缓存；读请求缓存未命中再回源数据库并回填。对强一致要求高的场景可加消息队列重试、延迟双删、版本号或 Canal 订阅 binlog。

2 从 0 到 1 搭建项目怎么设计架构？

先明确业务边界、用户角色、核心流程和非功能指标，再设计模块、数据模型、接口协议、权限、日志、部署和监控。初期避免过度设计，但保留扩展点。

3 AI 系统架构分哪些层？

可以分为接入层、业务层、AI 编排层、模型适配层、知识库层、工具层、数据层、观测与治理层。

4 门店 AI 数字化系统怎么设计？

场景包括门店问答、员工助手、销售话术、库存查询、客户画像、报表分析和工单处理。架构上接入 POS/CRM/库存/会员系统，用 RAG 处理制度和知识，用 Agent 调工具完成业务动作。

5 技术架构、模块、数据流、选型？

前端可用 Vue/React，后端用 NestJS/Java/Spring，数据用 MySQL/Redis/ES/向量库，AI 用模型网关统一适配。数据从业务系统同步到数据层和知识库，用户请求经过权限、意图识别、检索和工具调用后返回结果。

6 AI 对研发工作流冲击，团队有没有 AI 工作流？

有。需求阶段用 AI 辅助梳理 PRD 和边界，设计阶段生成方案和接口草稿，开发阶段辅助编码和单测，评审阶段让 AI 做初步代码检查，测试阶段生成用例和自动化脚本。

7 如何把 AI 融入开发流程？

把 AI 放进需求分析、技术方案、代码生成、重构、测试、文档和排障流程，但关键决策、架构边界、安全和代码合并仍由开发负责。

8 有 AI 编程规范吗？

可以建立规范：AI 生成代码必须可解释、可测试、符合项目风格；不能直接提交未审查代码；涉及安全、权限、支付、数据迁移必须人工复核。

9 AI 写的代码不可控，Code Review 怎么做？

AI 代码和人工代码一样走 PR、Review、CI、单测和静态检查。重点看边界条件、异常处理、安全、性能、依赖引入和是否破坏现有架构。

10 SSE 断线续传怎么做？

服务端给每条事件带 id，客户端断线重连时通过 Last-Event-ID 或自定义参数告诉服务端上次收到的位置。服务端需要保存一段时间的消息缓冲或任务状态，重新连接后从断点继续推送。

2 政府项目只能用国内工具影响？

4 同模型不同工具/同工具不同模型效果差异？

有差异。同模型在不同工具中受系统 Prompt、上下文检索、工具权限和交互设计影响；同工具切换模型则会影响推理、代码质量、结构化输出和稳定性。

5 AI 提效百分比？

39 题
1 CSS3 常用哪些？

Flex、Grid、transform、transition、animation、渐变、阴影、圆角、媒体查询、变量、粘性定位等。

2 渐变、旋转、缩放、动画了解吗？

渐变用 linear-gradient/radial-gradient，旋转缩放用 transform: rotate/scale，动画用 transition 或 @keyframes animation。

3 CSS 和 JS 动画怎么做？

CSS 适合简单状态变化和关键帧动画；JS 适合复杂交互、动态路径、物理效果和需要精确控制时间线的动画。

4 CSS 动画和 JS 动画区别？

CSS 声明式、性能好、实现简单；JS 控制力强但需要注意主线程性能。涉及 transform/opacity 的动画一般更流畅。

5 Canvas 用过吗？

用于图表、签名、图片编辑、粒子动画、游戏、海报生成等场景。它是像素级绘制，适合大量图形或自定义渲染。

6 Canvas 做过什么场景？

可以讲海报生成、签名板、数据大屏、红包动画、图片裁剪等，说明绘制、事件坐标、导出和性能优化。

7 红包/福袋动画怎么做？

简单的用 CSS 动画和序列帧，复杂的用 Canvas/Lottie。打开动作可通过缩放、抖动、光效、粒子、阴影和状态切换组合。

8 打开红包是画出来还是 UI 图？

取决于效果。静态主体通常用 UI 图保证还原度，动态部分用 CSS/Canvas/Lottie 叠加。

9 打开、抖动、浮动、阴影、缩放？

可以用 @keyframes 控制 transform 和 opacity，阴影用 box-shadow/filter，浮动用 translateY 循环，打开用 scale/rotate/clip-path 或序列帧。

10 WebSocket 前端连接流程？

创建 new WebSocket(url)，监听 open/message/error/close，连接成功后按协议发送鉴权或订阅消息。

11 WebSocket 怎么监听消息？

通过 socket.onmessage 或 addEventListener('message')，收到后解析 JSON，根据事件类型分发到对应处理逻辑。

12 怎么和后端约定事件？

约定统一消息格式，如 { event, requestId, data, timestamp }，并定义鉴权、订阅、心跳、错误码和重连后的补偿机制。

13 心跳/保活？

前端定时发送 ping，后端返回 pong；超过时间未收到响应则认为断开并重连。

14 断开怎么重连？

监听 close/error 后触发重连，限制最大次数，使用指数退避，重连成功后重新鉴权和订阅。

15 连接不上？

排查网络、URL、协议 ws/wss、鉴权、跨域、代理、服务端连接数和防火墙。前端要提示用户并走降级。

16 连接不上兜底？

可以降级 HTTP 轮询或让用户手动刷新，同时保留最后数据状态和错误提示。

17 是否降级 HTTP 轮询？

实时性要求不是极高时可以降级轮询，保证基本可用。

18 轮询频率怎么控制？

根据业务实时性设置，前台高频、后台低频，失败时降低频率，页面隐藏时暂停或延长间隔。

19 指数退避？

重连间隔从 1s、2s、4s、8s 增加，并设置最大值和随机抖动，避免大量客户端同时重连。

20 弱网怎么提示？

显示“连接中/重连中/数据可能延迟”，禁用关键操作或给出重试按钮，恢复后自动刷新。

21 实时数据不准确怎么提示？

展示最后更新时间、连接状态、数据延迟提示，必要时提供手动刷新和冲突提示。

22 ES6 常用特性？

let/const、箭头函数、模板字符串、解构、扩展运算符、Promise、class、module、Set/Map、async/await。

23 柯里化函数？

把接收多个参数的函数转换成一系列接收单个或部分参数的函数，便于参数复用和函数组合。

24 ES5 和 ES6 构造函数怎么写？

ES5 用函数构造器和 prototype；ES6 用 class 和 constructor，本质仍基于原型。

25 ES6/ES5 构造对象？

ES6 用 class/new，ES5 用构造函数/new 或 Object.create。

26 平时区分 ES5 和 ES6？

开发现代项目主要写 ES6+，通过 Babel/构建工具按目标浏览器降级。

27 Promise、扩展运算符等？

Promise 处理异步，扩展运算符用于数组/对象展开，箭头函数简化函数并绑定词法 this，模板字符串便于拼接。

28 浏览器兼容？

做过的话讲目标浏览器、Babel、PostCSS、polyfill、CSS 前缀和降级策略。

29 兼容注意点？

CSS 关注布局、前缀、单位和属性支持；JS 关注 API、语法、Promise、URLSearchParams、IntersectionObserver 等，需要 polyfill。

30 低版本打包降级问题？

常见是依赖包未转译、polyfill 缺失、目标浏览器配置不对。解决是配置 browserslist、transpileDependencies 和 core-js。

31 低版本路由变了页面不跳？

可能是 History API 不支持、hash/history 模式配置问题、polyfill 缺失或路由守卫异常。可改 hash 模式或加兼容处理。

32 框架是否抹平兼容？

框架抹平了一部分状态和渲染差异，但浏览器 API、CSS 支持和第三方依赖兼容仍要自己处理。

33 React 多还是 Vue 多？

按真实经验回答，再强调两个框架核心理念都理解，可以根据团队技术栈切换。

34 Vue2 和 Vue3 区别？

Vue2 用 Options API 和 defineProperty 响应式；Vue3 引入 Composition API、Proxy、Fragment、Teleport、Suspense 和更好的 TS 支持。

35 响应式原理？

Vue2 通过 Object.defineProperty 劫持 getter/setter；Vue3 通过 Proxy 代理对象，配合 track/trigger 收集和触发依赖。

36 defineProperty 和 Proxy 区别？

defineProperty 只能劫持已有属性，需要递归；Proxy 可以代理整个对象，能监听新增删除、数组索引等，能力更完整。

37 Vue3 性能优化？

编译时静态提升、PatchFlag、Block Tree、Proxy 惰性代理、更小运行时和更好的 Tree-shaking。

38 Vue2 数组/对象变更不触发？

新增对象属性、直接改数组索引或 length 可能无法触发。需要 Vue.set、this.$set 或使用被重写的数组方法。

39 数据变了视图没变原因？

可能是响应式未建立、直接修改嵌套属性、数组索引修改、key 不正确、组件缓存、异步更新未完成或状态源不是同一个。

12 秒杀系统架构？

前端限流和验证码，网关限流，Redis 预扣库存，消息队列削峰，后端异步创建订单，MySQL 最终落库，超时未支付释放库存。

13 高并发避免超卖？

核心是库存扣减必须原子化。可用 Redis Lua 原子扣减、MySQL 条件更新、乐观锁或队列串行化处理。

14 MySQL 怎么实现库存扣减？

使用 update sku set stock = stock - 1 where id = ? and stock > 0，检查影响行数。配合事务创建订单，失败则回滚。

15 Redis 分布式锁或 Lua？

分布式锁用于控制同一资源并发，但秒杀更推荐 Lua 原子判断和扣减库存，减少锁竞争。Lua 脚本在 Redis 单线程内原子执行。

16 限流、削峰、事务、乐观锁？

限流在网关和接口层，削峰用 MQ，事务保证订单和支付状态一致，乐观锁通过版本号或条件更新避免并发覆盖。

17 不用 Redis 只用 MySQL？

用条件更新、事务、唯一索引、防重复订单、乐观锁和队列控制并发。性能上限较低，需要控制流量。

18 微信支付接入流程？

19 支付后积分/虚拟产品到账？

以支付回调和主动查单结果为准，更新订单状态后发放权益。发放要幂等，防止重复回调导致重复到账。

20 支付为什么用异步队列？

支付回调要快速响应，权益发放、通知、积分流水等可异步处理，提升稳定性并削峰。

21 队列挂了怎么保证一致？

支付订单状态先落库，队列失败要可重试；定时任务扫描已支付未发放订单做补偿，任务处理要幂等。

22 回调验签流程？

获取回调头和报文，使用平台证书验证签名，解密资源数据，校验商户号、订单号、金额和状态，再更新本地订单。

23 MCP/Skill 如何辅助开发？

常见说法是 Data Mapper 和 Active Record。Data Mapper 通过 Repository 操作实体，Active Record 实体自己带保存查询方法。

10 TypeORM 查询慢怎么抓 SQL？

开启 logging，或用 QueryBuilder 的 getSql()，再结合 MySQL slow query log、EXPLAIN、索引分析和执行计划调优。

11 PostgreSQL 物化视图？

物化视图会把查询结果持久化，适合复杂统计查询加速。数据不是实时的，需要 refresh。

12 MySQL 慢查询优化？

开慢查询日志，定位 SQL，用 EXPLAIN 看索引、扫描行数、临时表、排序；优化索引、改写 SQL、分页方式、减少回表和大事务。

13 分库分表/读写分离经验？

按真实经验回答。没有深度经验可以说了解方案和代价：分库分表解决容量和写入压力，读写分离解决读压力，但会带来事务、Join、分页和一致性复杂度。

14 Bull Queue 进程崩溃/K8S 重启任务会丢吗？

Bull 基于 Redis 持久化任务，正常配置下不会因为 worker 崩溃就丢。重启后 stalled job 会被重新认领，但任务本身必须幂等。

15 如何保证队列任务不丢和一致性？

任务入队前后落库记录状态，开启重试和死信队列，worker 幂等处理，定时扫描卡住任务补偿，关键状态变更用事务。

16 重复推理优化、降成本？

使用请求指纹去重、结果缓存、任务幂等键、相同文档 Embedding 复用、失败重试次数限制和模型分层。

17 任务中断断点续跑？

把长任务拆成步骤，每一步记录状态、输入、输出和进度。重启后从最近成功步骤继续，已完成步骤不重复执行。

18 前端感知任务状态？

后端维护任务状态机，前端通过 SSE/WebSocket/轮询获取 pending/running/failed/done/progress，并展示重试或取消。

19 SSE 和 WebSocket 区别？

SSE 单向、基于 HTTP、适合服务端推流；WebSocket 双向、实时性更强，适合频繁双向交互。

20 AGUI 动态渲染表单/卡片流程？

后端输出结构化 UI 协议，前端根据组件类型、字段、校验、布局和事件映射到本地组件库，动态渲染并回传用户操作。

21 流式协议前端边接收边渲染？

用 SSE/fetch stream 读取 chunk，按协议解析事件，增量更新消息、表单或卡片状态。注意半包、顺序、错误事件和结束标记。

22 协议底层 API 设计逻辑？

核心是事件驱动：定义 message、delta、tool_call、ui_update、error、done 等事件类型，每个事件有唯一 ID 和 payload，前端按类型分发。

23 数据库事务？

事务是一组操作的原子执行单元，满足 ACID：原子性、一致性、隔离性、持久性。

24 必须用事务的场景？

例如支付成功后更新订单、积分流水和权益发放记录；或者创建任务时同时写任务表、步骤表和日志表，任何一步失败都要回滚。

25 LangChain 不同版本/模块区别？

新版本更模块化，拆成 core、community、具体 provider 包等；LangGraph 负责图状态编排，LangSmith 负责观测评估。

26 多智能体动态路由？

通过意图分类、规则、模型判断或状态条件，把任务分配到不同 Agent。LangGraph 中可以用 conditional edge 根据状态返回下一个节点。

27 查询改写？

查询改写是把用户原始问题改成更适合检索的问题，比如补全上下文、提取关键词、生成多个子查询，提升召回率。

28 知识库用什么向量库？

按真实情况回答。可补充选型依据：数据规模、过滤、混合检索、部署成本和团队熟悉度。

29 ES + 向量库混合检索流程？

用户问题同时走关键词检索和向量检索，合并结果后去重、归一化打分、按权限和元数据过滤，再重排后进入模型。

30 为什么还要 Re-rank？

初召回追求覆盖，排序不一定精准。Re-rank 用更强的交叉编码器或模型理解 Query 与文档关系，提高最终上下文质量。

31 领域词典？

领域词典用于同义词、专业术语、缩写和业务实体归一化，可提升关键词检索、查询改写和意图识别效果。

32 Vue2 数组方法如何重写？

Vue2 拦截 push/pop/shift/unshift/splice/sort/reverse，在调用原生方法后手动通知依赖更新，并对新增元素做响应式处理。

33 哪些数组方法无法原生拦截？

直接通过索引赋值和修改 length 无法被 defineProperty 拦截，需要 Vue.set 或 splice。

34 Vue3 Proxy 优势？

能监听新增、删除、索引、in、ownKeys 等操作，响应式更完整，初始化时也更惰性。

35 Vue3 WeakMap 作用？

Vue3 用 WeakMap 存 target 到依赖 Map 的映射，弱引用避免对象不用后无法被垃圾回收。

36 Harness 概念？

Harness 通常指测试或执行“脚手架/运行框架”，用于提供可控环境、Mock、断言和执行入口。具体含义要结合上下文，若是 CI/CD 产品 Harness，则是持续交付和流水线平台。
